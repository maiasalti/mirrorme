# MirrorMe Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A Chrome extension + companion web app that composites clothing from any store's product page onto the user's own photo via Gemini 2.5 Flash Image, with accounts, private photo storage, try-on history, and chaining.

**Architecture:** Thin MV3 extension (Vite + CRXJS + React + TS) captures garment image URLs (click-to-select first, auto-detect on top) and calls a Next.js (App Router) backend. The backend verifies the Supabase JWT, enforces a quota checkpoint, fetches base-photo + garment bytes server-side (SSRF-guarded), calls `gemini-2.5-flash-image` via `@google/genai`, stores results in a private Supabase Storage bucket, and returns short-lived signed URLs. Postgres with RLS everywhere; Gemini + service-role keys never leave the server.

**Tech Stack (versions verified against live docs/npm 2026-06-11):**
- `@google/genai` ^2.8.0, model `gemini-2.5-flash-image` (GA "nano banana", ~$0.039/image)
- `@supabase/supabase-js` ^2.108, `@supabase/ssr` ^0.12, Supabase CLI via `npx supabase` (no Docker needed for hosted `db push`)
- Next.js 16 (App Router, `src/` dir, Tailwind v4), deploy target Vercel
- `@crxjs/vite-plugin` ^2.5.0 + Vite 7 + React 19 + TypeScript; MV3
- Vitest for unit tests (SSRF guard, data-URL parsing, srcset/auto-detect logic)

## Verified third-party API contracts (do not deviate)

**Gemini multi-image call** (docs: ai.google.dev/gemini-api/docs/image-generation):

```ts
import { GoogleGenAI } from '@google/genai'
const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY })
const response = await ai.models.generateContent({
  model: 'gemini-2.5-flash-image',
  contents: [
    { text: PROMPT },
    { inlineData: { mimeType: 'image/jpeg', data: base64Person } },
    { inlineData: { mimeType: 'image/png', data: base64Garment } },
  ],
})
// result image: first part with part.inlineData in response.candidates[0].content.parts
// part.inlineData.data = base64, part.inlineData.mimeType = 'image/png'
```
Limits: 20MB total request; inputs must be image/png|jpeg|webp|heic|heif; keep ≤3 input images.

**Supabase server JWT verification:** `admin.auth.getClaims(jwt)` (validates signature; falls back to a server call on HS256 projects). User id = `data.claims.sub`.

**Extension Google sign-in** (official Supabase docs, "Chrome Extensions" tab of the Google login guide): Google OAuth client of type *Chrome Extension*; `chrome.identity.launchWebAuthFlow` to `https://accounts.google.com/o/oauth2/auth` with `response_type=id_token` and redirect `https://<ext-id>.chromiumapp.org/`; parse `id_token` from the redirect hash; `supabase.auth.signInWithIdToken({ provider: 'google', token: idToken })`. Requires the Google client id registered in Supabase's Google provider "Client IDs" list and (if nonce errors) "Skip nonce check" enabled. Stable extension id in dev via manifest `key`.

**CRXJS dynamic content script:** `import selectScript from './content/select?script'` in the background, then `chrome.scripting.executeScript({ target: { tabId }, files: [selectScript] })` — bundled by CRXJS, works with `activeTab` + `scripting` permissions only, no `content_scripts` manifest entry, no broad host permissions.

**MV3 fetch rules:** popup/background can fetch cross-origin only for origins in `host_permissions` → list the API origin (`http://localhost:3000/*` in dev, prod URL via env at build) and the Supabase project URL. The injected content script can NOT fetch cross-origin — it only resolves URLs/data-URLs and messages them back.

  email text,

alter table public.profiles enable row level security;
create policy "profiles_select_own" on public.profiles
  for select to authenticated using ((select auth.uid()) = id);

-- auto-create profile on signup
create or replace function public.handle_new_user()
returns trigger language plpgsql security definer set search_path = public as $$
begin
  insert into public.profiles (id, email) values (new.id, new.email)
  on conflict (id) do nothing;
  return new;
end; $$;
create trigger on_auth_user_created
  after insert on auth.users
  for each row execute function public.handle_new_user();

create table public.photos (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references public.profiles (id) on delete cascade,
  storage_path text not null,
  is_default boolean not null default false,
  created_at timestamptz not null default now()
);
alter table public.photos enable row level security;
create policy "photos_all_own" on public.photos
  for all to authenticated
  using ((select auth.uid()) = user_id) with check ((select auth.uid()) = user_id);

create table public.tryons (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references public.profiles (id) on delete cascade,
  base_photo_id uuid references public.photos (id) on delete set null,

);
alter table public.tryons enable row level security;
create policy "tryons_select_own" on public.tryons
  for select to authenticated using ((select auth.uid()) = user_id);
-- tryon inserts happen server-side only (service role bypasses RLS)

-- atomic quota counter (called server-side after each successful generation)
create or replace function public.increment_tryon_count(p_user_id uuid)
returns void language sql security definer set search_path = public as $$
  update public.profiles set tryon_count = tryon_count + 1 where id = p_user_id;
$$;

-- private buckets
insert into storage.buckets (id, name, public)
values ('photos', 'photos', false), ('generated', 'generated', false)
on conflict (id) do nothing;

-- storage RLS: users manage only their own folder in `photos`
create policy "photos_bucket_insert_own" on storage.objects
  for insert to authenticated

create policy "photos_bucket_select_own" on storage.objects
  for select to authenticated

create policy "photos_bucket_delete_own" on storage.objects

```ts
import { createAdminClient } from './supabase/admin'
import { createClient } from './supabase/server'

export async function getUserId(req: Request): Promise<string | null> {
  const header = req.headers.get('authorization')
  if (header?.startsWith('Bearer ')) {
    const jwt = header.slice(7)
    const { data, error } = await createAdminClient().auth.getClaims(jwt)
    if (error || !data?.claims?.sub) return null
    return data.claims.sub
  }
  const supabase = await createClient()
  const { data } = await supabase.auth.getClaims()
  return data?.claims?.sub ?? null
}
```

```ts
import { GoogleGenAI } from '@google/genai'

const MODEL = 'gemini-2.5-flash-image'

const BASE_RULES =
  'Keep the person\'s face, hair, skin tone, body shape, pose, and the background exactly as they are in the first image. ' +
  'Preserve the garment\'s exact color, pattern, texture, logos, and details from the second image. ' +
  'Fit the garment naturally and realistically to the person\'s body and pose, with correct draping, lighting, and shadows. ' +
  'Output a photorealistic image.'

export function tryOnPrompt(chained: boolean): string {
  return chained
    ? 'The first image shows a person already wearing one or more clothing items from previous try-ons. ' +
      'ADD the clothing item from the second image to their outfit, layering it naturally. ' +
      'Keep every previously applied garment visible and unchanged unless the new garment necessarily covers it. ' + BASE_RULES
    : 'Take the person in the first image and the clothing item in the second image. ' +
      'Generate an image of the same person wearing that clothing item. ' + BASE_RULES
}

export async function generateTryOn(opts: {
  base: { data: string; mimeType: string }
  garment: { data: string; mimeType: string }
  chained: boolean
}): Promise<{ data: Buffer; mimeType: string }> {
  const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY! })
  const response = await ai.models.generateContent({
    model: MODEL,
    contents: [
      { text: tryOnPrompt(opts.chained) },
      { inlineData: { mimeType: opts.base.mimeType, data: opts.base.data } },
      { inlineData: { mimeType: opts.garment.mimeType, data: opts.garment.data } },
    ],
  })
  const part = response.candidates?.[0]?.content?.parts?.find((p) => p.inlineData)
  if (!part?.inlineData?.data) throw new Error('Gemini returned no image')
  return { data: Buffer.from(part.inlineData.data, 'base64'), mimeType: part.inlineData.mimeType ?? 'image/png' }
}
```
